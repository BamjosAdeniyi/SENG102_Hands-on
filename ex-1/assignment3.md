## ANSWER

### 1. Concept

Semantic HTML elements are simply elements with meaning. The use and definition of semantic HTML elements in the code tells the browser and the developer what they are supposed to do. In simpler words, these elements describe the type of content they are supposed to contain. Semantic elements provide meaningful tags that clearly describe their purpose, thereby improving readability and accessibility.

### 2. Importance of Semantic HTML

- It clearly defines the role and content of elements.
- It improves code readability and structure.
- It enhances accessibility for screen readers.
- It helps browsers and search engines understand page content.
- Easier maintenance.
- Improves accessibility.
- Cleaner and more readable code.
- Easier maintenance.

### 3. Semantic structure elements

- **`<article>`**: Used for content that stands alone and can be independently distributed or reused, such as a blog post or news article.
- **`<aside>`**: Used to place content in a sidebarâ€”aside from the existing content.
- **`<details>`**: Defines additional details that the user can hide or view; `<summary>` defines a visible heading for a `<details>` element.
- **`<figure>` / `<figcaption>`**: Used together to display an image (or illustration) with a descriptive caption.
- **`<footer>`**: For the footer located at the bottom of any article or document; can contain contact details, copyright information, etc.
- **`<form>`**: (description missing in original) Used to collect user input.
- **`<header>`**: Used for the header of a section or page, typically introductory content.
- **`<main>`**: Defines the main content of the document. The content inside the `<main>` tag should be unique.
- **`<mark>`**: Used to highlight text.
- **`<nav>`**: Defines a set of navigation links, such as a navigation bar or menu.
- **`<table>`**: Used to create a table.
- **`<section>`**: Used to split a page into different sections.
